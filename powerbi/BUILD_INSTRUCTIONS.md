# Power BI Build Instructions

This document is a step-by-step guide to building the dashboard in Power BI Desktop. Follow it end-to-end and you will have a working report in roughly 60-90 minutes.

## Pre-requisites

- Power BI Desktop installed (free)
- The two CSV files generated by the scripts:
  - `data/suppliers_master.csv` (100 suppliers with raw metrics)
  - `data/supplier_narratives.csv` (AI-generated narratives joined by supplier_id)
- A Power BI license (free tier is enough for this build; Pro tier needed for "Publish to Web" sharing)

## Step 1: Load the Data

1. Open Power BI Desktop
2. **Home** → **Get data** → **Text/CSV**
3. Load `suppliers_master.csv` and `supplier_narratives.csv`
4. In the Power Query Editor:
   - Verify column types: numeric columns should be Decimal Number or Whole Number
   - `single_source` should be True/False
   - Click **Close & Apply**

## Step 2: Set Up Relationships

1. **Model view** (left sidebar)
2. Drag `supplier_id` from `suppliers_master` to `supplier_id` in `supplier_narratives`
3. Verify cardinality: One-to-One
4. Cross filter direction: Both

## Step 3: Create the DAX Measures

In the Data view, right-click `suppliers_master` → New measure. Add each measure from `dax_measures.md`. Start with these 5 essential ones:

```DAX
Total Suppliers = COUNTROWS(suppliers_master)

Avg Risk Score = AVERAGE(suppliers_master[overall_score])

Red Suppliers = CALCULATE(COUNTROWS(suppliers_master), suppliers_master[tier] = "RED")

Yellow Suppliers = CALCULATE(COUNTROWS(suppliers_master), suppliers_master[tier] = "YELLOW")

Green Suppliers = CALCULATE(COUNTROWS(suppliers_master), suppliers_master[tier] = "GREEN")
```

Add the remaining measures from `dax_measures.md` as you build the visuals that need them.

### Visuals (build in this order)

1. **4 Card visuals** at the top: Total Suppliers, RED, YELLOW, GREEN
   - Format: Big number, conditional color (red/yellow/green for the tier cards)

2. **Stacked Bar Chart** (middle):
   - Y-axis: `region`
   - X-axis: count of suppliers
   - Legend: `tier`
   - Colors: RED=#dc2626, YELLOW=#eab308, GREEN=#16a34a

3. **Top 10 At-Risk Table** (bottom):
   - Add a Table visual
   - Columns: `supplier_id`, `supplier_name`, `region`, `category`, `overall_score`, `tier`, `narrative`, `recommended_action`
   - Sort by `overall_score` ascending
   - Filter to Top N: top 10 by `overall_score` ascending
   - Conditional formatting on `tier`: background color matches tier

4. **Slicers** (top right): `region`, `category`, `tier`. Lets PM filter the whole page interactively.

### How to make this a drill-through page

1. Right-click the page tab → **Rename** to "Supplier Detail"
2. In the Visualizations pane, find **Drillthrough**
3. Drag `supplier_id` from `suppliers_master` into the **Drill through** well
4. Now from Page 1, right-clicking any supplier in the Top 10 table will show "Drillthrough → Supplier Detail"

### Visuals on this page

1. **Card** for supplier name
2. **Card** with conditional color for tier
3. **Multi-row card** for region/category/spend/relationship
4. **Bar chart** (or radar if you have a custom visual) for the 5 dimension scores
5. **Multi-row card** for underlying metrics
6. **Card** with the narrative text (use the `narrative` column directly)
7. **Card** for recommended_action

## Step 6: Page 3 - Q&A Page

This is where Power BI Copilot or the Q&A visual lives.

### If you have Power BI Copilot:

1. Add a Copilot visual to the page
2. Suggested prompts to seed:
   - "Show me APAC suppliers with quality escapes greater than 2"
   - "Which suppliers have PPAP approved but OTD below 85%?"
   - "Total annual spend at-risk for RED-tier suppliers?"

### If you don't have Copilot (free tier):

1. Add a **Q&A** visual (standard Power BI visual)
2. Configure suggested questions in the Q&A setup
3. The Q&A visual works with natural language but is more limited than Copilot

Either way, this page demonstrates the AI-augmented exploration capability.

## Step 7: Polish

1. **Apply a theme**: View → Themes → pick a clean one (Innovate, Executive, or import a custom theme)
2. **Page titles**: each page should have a consistent header
3. **Tooltips**: hover-over tooltips for the bar chart should show the supplier list
4. **Bookmarks**: optional, but a "RED only" bookmark and "GREEN only" bookmark are quick wins

## Step 8: Publish

1. **Home** → **Publish** → My Workspace
2. Open the report in Power BI Service (browser)
3. **File** → **Embed report** → **Publish to web (public)**
4. Copy the embed link - this is the demo URL for your resume

**Important**: Publish to Web makes the report public. This is fine for synthetic data, never use it with real data.

## Common Issues

### "I don't see Copilot"

Power BI Copilot requires a Pro license tenant with Copilot capacity. Free tier users will not have it. The Q&A visual is the supported alternative on free tier.

### "My measures show errors"

Check that column types are correct. Power BI will silently treat numeric columns as text if the CSV has any non-numeric value. Re-import via Power Query and explicitly set types.

### "Drill-through doesn't work"

Verify the supplier_id field is in the Drillthrough well on the destination page, AND the field exists on the source page in the table you're right-clicking.
